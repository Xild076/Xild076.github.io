Big Idea 1: Creativity (Creative Development)

Summary

Computing is a creative activity. This Big Idea emphasizes designing and creating computational artifacts (like programs, images, audio, videos, or web content) through an iterative, often collaborative process. Creativity in computer science includes creative problem-solving and expression in any program or digital artifact.

Key Concepts & Terms

The Creative Development Process is an iterative design approach for creating programs or artifacts. It's often exploratory and can involve brainstorming, storyboarding, trial-and-error, and rapid prototyping. An iterative design approach for creating programs or artifacts. It may include steps like ideation â†’ prototype â†’ test â†’ refine. There is no single "right" method - it can involve trial and error and exploring personal interests.

A Computational Artifact is anything created by a human using a computer. This could be software, digital music, data visualizations, websites, 3D models, etc., created for expression or problem-solving. Any product of human creativity made with a computer - e.g. software, images, music, videos, or web pages.

Collaboration in creative computing involves working with others, allowing multiple perspectives and skills to influence a project, often leading to more innovative solutions. Effective collaboration involves communication and division of tasks. Working with others to produce a computational outcome. Effective collaboration uses communication and often online tools. It can enhance creativity by bringing in multiple viewpoints and expertise.

Iteration means repeating steps, revising, and improving a project or algorithm. It's key to refining artifacts by continually testing and debugging. Repeating steps, revising, and improving a project or algorithm. Iteration is key to refining artifacts (e.g., continually testing and debugging a program to add features or fix flaws).

Example

Imagine creating a mobile app to track homework. You'd first brainstorm features (like reminders or a calendar). Next, you prototype a simple version. You test it, gather feedback (e.g., users find the interface confusing), then iterate â€“ redesigning it. You might collaborate with a classmate where one handles front-end design and the other back-end logic. Together, you produce a better app. This illustrates Big Idea 1: creativity and iteration leading to an improved computational artifact.

Big Idea 2: Abstraction

Summary

Abstraction reduces complexity by hiding details to focus on essential concepts. In computing, abstraction allows us to represent complex things more simply, using layers and models. It appears in data representation (binary to images), generalization in code (functions), and the layered architecture of computing systems (hardware to software).

Key Concepts & Terms

Abstraction (General): The concept of simplifying complexity by focusing on main ideas and hiding unnecessary detail. Simplifying complexity by focusing on main ideas and hiding detail. In computing, this could mean representing complex data in a simplified form or structuring programs in layers.

Data is represented at the lowest level by Bit (binary digits: 0s and 1s). Higher-level abstractions group bits. The most basic unit of digital data, a binary digit 0 or 1. All digital information is composed of bits in sequence (bit streams).

Number systems like Binary (Base-2), Decimal (Base-10), and Hexadecimal (Base-16) are different abstractions for representing numbers. Hexadecimal is often a human-friendly shorthand for binary. Binary (Base-2): A number system using only 0s and 1s. Example: decimal 13 is binary 1101. Hexadecimal (Base-16): Uses 16 symbols (0-9, A-F) and is often used as a compact representation of binary groups (e.g., one hex digit represents four bits).

ASCII/Unicode are character encoding standards that map numbers to text characters, allowing text to be represented in binary. Character encoding standards (abstractions) that map numerical values to characters. ASCII is an older standard, Unicode is more comprehensive supporting characters from most world languages. E.g., in ASCII, decimal 65 represents 'A'.

Logic Gate: A hardware abstraction performing basic Boolean operations (AND, OR, NOT), forming building blocks of digital circuits. A hardware abstraction that performs a basic boolean operation (AND, OR, NOT, etc.). Gates are the building blocks of digital circuits, modeled by Boolean algebra.

Models and Simulations are abstractions of real-world phenomena, simplifying complex systems to make them easier to understand, experiment with, or make predictions. Model: A simplified representation of something complex (e.g., a mathematical model of population growth), omitting extraneous details. Simulation: The use of a computer model to imitate a real-world process over time, allowing experimentation.

Procedures/Functions in programming are abstractions that encapsulate a block of code to perform a specific task and can be reused. Named blocks of code that perform a specific task. They help manage complexity by allowing code reuse and hiding the details of how a task is done behind a function name.

Parameters are inputs to procedures/functions that allow them to generalize behavior for different data. Inputs to a procedure that allow it to generalize behavior. For instance, `drawSquare(size)` uses `size` as a parameter so it can draw squares of any side length.

Example of Layered Abstraction

Reading an email involves many layers: 1. You see text/images in your email client (high-level app). 2. The email program calls OS functions to fetch data. 3. Data travels as packets over the Internet (bits routed via protocols). 4. Wi-Fi router turns bits into radio signals. 5. Email text is encoded (ASCII/UTF-8). At each step, lower-level details are hidden, making the complexity manageable.

Big Idea 3: Data and Information

Summary

Data and information facilitate the creation of knowledge. This Big Idea explores how we collect, process, analyze, and visualize data to gain insights. Computing enables handling large datasets, finding patterns, and deriving meaning. The cycle is often Data â†’ Information â†’ Knowledge.

Detailed Concepts

Data Collection and Generation

Data comes from sensors, transactions, surveys, user content, etc. Modern tech generates Big Data. Challenges include volume, cleaning, and privacy.

Extremely large data sets (e.g., billions of records) that require specialized tools and techniques for analysis due to their volume, velocity, or variety. Analyzing big data can reveal complex patterns and trends.

Data vs. Information vs. Knowledge

Data: Raw facts, figures (e.g., a list of numbers). Information: Processed, organized, meaningful data (e.g., an average). Knowledge: Interpreted information leading to understanding (e.g., "this month was hotter").

Data: Raw values or facts without context. Information: Data processed or organized to have meaning. Knowledge: Insights or understanding gained from information.

Processing and Cleaning Data

Raw data is often messy. Processing includes Cleaning Data (fixing errors, normalizing formats), Filtering Data (isolating relevant subsets), combining sources, Clustering (grouping similar data), and Classification (assigning labels).

Cleaning Data: Correcting or removing inaccurate records, standardizing formats to prepare raw data for analysis. Filtering Data: Selecting a subset of data based on criteria. Clustering: Grouping data points with similar attributes (unsupervised). Classification: Assigning data to predefined categories based on features (supervised).

Visualization and Communication

Visualization (charts, graphs) helps humans understand complex data. Choosing the right type and clear labeling are crucial. Interactivity enhances communication.

A graphical representation of information (charts, graphs, plots) to help humans understand data patterns and trends.

Large Data Sets ("Big Data")

Enable discovery but pose challenges in storage, processing power, and privacy. Specialized tools (databases, MapReduce) are used. Metadata (data about data, e.g., timestamps, source) is crucial for context and filtering.

Data about data. Examples: a photo's metadata might include date taken, camera type, location; a data table's metadata might be column descriptions or data source info. Metadata increases the effective use of data by providing context.

Trade-offs: Storage, Security, Compression

Decisions involve balancing factors:

  • Storage vs. Precision: Fewer bits save space but reduce accuracy.
  • Privacy vs. Utility: Data use benefits vs. protecting individuals.
  • Security vs. Convenience: Strong security can be less convenient.
  • Lossless Compression vs. Lossy Compression: Lossless perfectly reconstructs data (e.g., PNG, ZIP) with moderate size reduction. Lossy permanently discards some data for much smaller sizes (e.g., JPEG, MP3), trading quality for size.

Lossless Compression: Allows perfect reconstruction of original data from compressed data (no information lost). Lossy Compression: Discards some data deemed less important, so original cannot be exactly reconstructed; achieves higher compression by accepting some quality loss.

Other Key Terms

Data Mining: Examining large databases to find hidden patterns or correlations. Techniques for examining large pre-existing databases to generate new information, often involving looking for hidden patterns or correlations in big data.

Example: School Performance Analysis

A school collects data on student study hours, attendance, and grades (Data). They clean it and calculate average study hours for grade ranges (Information). A scatter plot reveals students with high absences often have lower grades (Visualization). The school deduces that promoting study habits and attendance could improve performance (Knowledge).

Big Idea 4: Algorithms

Summary

Algorithms are precise, unambiguous, finite sets of instructions for processes that can be executed by a computer. They are the core methods for solving problems or performing tasks. This Big Idea covers how algorithms are developed, expressed, evaluated (for efficiency, correctness, clarity), and the limitations of what algorithms can solve.

Key Concepts & Terms

An Algorithm is a finite set of instructions that accomplishes a specific task. It must be precise and unambiguous. A finite set of instructions that accomplishes a specific task. It must be precise, unambiguous, and guaranteed to terminate.

Building blocks of algorithms: Sequencing (executing steps in a given order), Selection (making decisions using conditionals like IF-ELSE), and Iteration (repetition using loops like FOR or REPEAT UNTIL). Sequencing: The execution of steps in order, one after another. Selection: Using conditionals to decide which steps to execute next (branching logic, e.g., IF statements). Iteration: Repeating steps, either a fixed number of times or until a condition is met (loops).

Algorithms can be expressed in natural language, Pseudocode (a structured, code-like but language-agnostic way to describe logic), or visual diagrams like flowcharts. A way to describe an algorithm using code-like language that isnâ€™t specific to any real programming language. Itâ€™s human-readable and captures the logic of the algorithm.

Algorithm Efficiency is evaluated by its time complexity (how runtime grows with input size) and space complexity (memory usage). "Reasonable time" algorithms are typically polynomial (e.g., linear O(n), quadratic O(nÂ²)), while "unreasonable time" algorithms are often exponential (e.g., O(2^n)) and become infeasible for large inputs. Often measured in terms of time complexity (how execution time grows with input size) and space complexity (memory usage). Polynomial time (e.g., O(n), O(nÂ²)) is generally considered reasonable; exponential time (e.g., O(2^n)) is generally unreasonable for large inputs.

A Heuristic is an approach to problem-solving that employs a practical method not guaranteed to be optimal or perfect, but sufficient for reaching an immediate, short-term goal or approximation when exact methods are too slow or complex. An approach to solving a problem that may not be perfect or exact but is practical for reaching a solution in a reasonable time. Often used for complex problems where exact algorithms are too slow (e.g., Traveling Salesman Problem).

A Decidable Problem is one for which an algorithm can be constructed to always produce a correct yes/no answer for all inputs. A problem for which an algorithm can be constructed to answer yes/no for all inputs (the algorithm will always terminate with a correct answer).

An Undecidable Problem is one for which no algorithm can be written that always leads to a correct yes/no answer for all possible inputs (e.g., the Halting Problem). A problem for which no algorithm can decide the answer for all cases (no single algorithm is guaranteed to always produce a correct yes/no answer for every possible input). The Halting Problem is a classic example.

Empirical Testing involves running an algorithm with various inputs to observe its behavior (e.g., timing, checking results) to analyze performance or correctness. Running an algorithm with various inputs to observe its behavior (e.g., timing how long it takes, checking results) as a way to analyze performance or correctness.

Linear Search checks items one by one (O(n) complexity). Binary Search repeatedly halves the search space (O(log n) complexity) but requires the list to be sorted. Linear Search: Checks each item in a list sequentially until the target is found or the list ends. Efficiency: O(n). Binary Search: Efficiently finds an item in a sorted list by repeatedly dividing the search interval in half. Efficiency: O(log n). Requires sorted data.

Example: Maze Solving

To solve a maze: An algorithm might involve "always turn right" (a heuristic, not guaranteed). Depth-first search systematically explores paths and backtracks (decidable for finite mazes). Breadth-first search finds the shortest path but might use more memory. These illustrate sequencing (follow corridor), selection (choose turn at intersection), and iteration (keep moving/backtracking).

Big Idea 5: Programming

Summary

Programming enables problem-solving, human expression, and knowledge creation. It's the process of designing, writing, testing, debugging, and maintaining the source code of computer programs. This Big Idea covers fundamental programming concepts, the development process, how programs implement algorithms, and how abstraction helps manage complexity.

Detailed Concepts

Program Purpose and Functionality

Programs are developed for creative expression, personal curiosity, creating new knowledge, or solving problems for individuals, organizations, or society. Programs can also have unexpected outcomes beyond their original purpose.

Program Development Process

  • Iterative Development: Write, test, debug, improve, add features, repeat. Programs are rarely perfect on the first try.
  • Incremental Development: Build programs in small, manageable, testable pieces (modules/functions), then integrate them.
  • Testing and Debugging: Continuously test with various inputs (including edge cases) to find and fix errors (Bugs). Debugging is a critical skill.
    • An error, flaw, or fault in a computer program or system that causes it to produce an incorrect or unexpected result, or to behave in unintended ways.
  • Documentation: Comments in code and external documentation explain the program's purpose and usage, aiding development and maintenance.
  • Collaboration: Coding in teams can reduce workload, bring diverse expertise, and improve error detection. Practices include pair programming and code reviews.

Implementing Algorithms

Programs execute algorithms. High-level code (Python, JavaScript) is translated into machine instructions. Understanding program flow (sequential, altered by loops/conditionals/function calls) is key.

Variables store data that can change. They are named storage locations, initialized with values, and can be updated.

A named storage location in a program that holds a value which may change during program execution (e.g., `score`, `userName`, `temperature`). In AP pseudocode: `x <- 5` assigns 5 to x.

Programs use memory (for variables/data) and a CPU (to execute instructions), and involve input/output.

Abstraction in Programs

Programming is facilitated by abstractions that manage complexity:

  • Procedures/Functions: Reusable blocks of code performing specific tasks. They encapsulate logic. A named block of code that performs a specific task. Defined once and can be called (invoked) multiple times. May have parameters and may return a value.
  • Parameters: Inputs to procedures that allow them to work with different data, generalizing solutions and avoiding code duplication. An input variable for a procedure/function, used to pass data into it so it can operate on that data. Parameters make procedures more flexible and reusable.
  • Return Values: Output from a function sent back to the caller. The output of a function that is sent back to the part of the program where the function was called. In AP pseudocode, `RETURN (value)` is used.
  • Data Abstractions: Structures like Lists (Arrays) organize data, simplifying management (e.g., one list of 100 items vs. 100 variables). A data structure that stores an ordered collection of items, which can be of mixed types. Elements are accessed by an index. In AP CSP, lists are 1-indexed.
  • Libraries/APIs: Collections of pre-written procedures for common tasks (e.g., graphics, data analysis). Using them is an abstraction. Library: A collection of pre-written code (functions, procedures, classes) that provides functionalities for common tasks, allowing programmers to reuse code. API (Application Programming Interface): A set of rules and protocols for building and interacting with software components; it defines how different software pieces can communicate.

Other Key Terms

Syntax: The set of rules defining correctly structured programs in a language. The set of rules that defines the combinations of symbols that are considered to be correctly structured programs in a specific programming language.

Conditional (Branch): Uses IF/ELSE to execute code based on whether a condition is true or false, enabling decision-making. A programming construct (e.g., IF, ELSE IF, ELSE) that allows a program to execute different sequences of statements based on whether a Boolean condition evaluates to true or false.

Event-driven programming: Program flow is determined by events (e.g., mouse clicks, key presses, messages from other programs). Common in GUIs. A programming paradigm in which the flow of the program is determined by events such as user actions (mouse clicks, key presses), sensor outputs, or messages from other programs/threads.

Runtime Error: An error occurring during program execution (e.g., division by zero, index out of bounds). An error that occurs during the execution of a program (e.g., division by zero, accessing an array element with an out-of-bounds index, trying to open a non-existent file). Often causes the program to crash or behave unexpectedly.

Logic Error: The program runs without crashing but produces incorrect results due to a flaw in the algorithm or implementation. An error in a program's source code that results in incorrect or unexpected behavior, even though the program may run without crashing. The syntax is correct, but the underlying algorithm or its implementation is flawed.

Example: Simple Game Development

To create a character-collects-coins game: 1. **Plan:** Outline movement, collision, scoring. 2. **Incremental Coding:** Get character moving, then add one coin and collision logic, then multiple coins (using a list for coin positions â€“ a data abstraction). Test at each step. 3. **Abstraction:** A function `checkCollision(character, coin)` returns true/false. A function `moveCharacter(direction)` updates position. 4. **Testing:** Test coin collection, movement boundaries. 5. **Documentation:** Add comments. This iterative, abstracting, testing approach improves correctness.

Big Idea 6: The Internet

Summary

The Internet is a global network of networks connecting millions of devices, built on openness, redundancy, and scalable design. It uses protocols like TCP/IP to enable communication. This Big Idea covers its structure, design principles (hierarchy, fault tolerance through redundancy, packet switching), scalability, and crucial cybersecurity issues.

Internet Structure & Protocols

Connects devices (computers, phones, servers, IoT) via Routers that forward data packets.

A networking device that forwards data packets between computer networks. Routers perform the traffic directing functions on the Internet.

Key Protocols (agreed-upon rules for communication):

A set of rules governing the exchange or transmission of data between devices. Examples: IP, TCP, HTTP, DNS.
  • IP (Internet Protocol): Addressing (IPv4, IPv6) and packet routing. Every device has an IP address.
    • The principal communications protocol in the Internet protocol suite for relaying datagrams across network boundaries. Its routing function enables internetworking, and essentially establishes the Internet. Defines addressing (IP addresses) and packet structure.
  • TCP (Transmission Control Protocol): Reliable, ordered data delivery (breaks data into packets, reassembles, retransmits if lost). Used for web, email.
    • A standard that defines how to establish and maintain a network conversation through which application programs can exchange data. TCP works with IP to send packets between computers over the Internet. It provides reliable, ordered, and error-checked delivery of a stream of octets.
  • UDP (User Datagram Protocol): Faster, but no guaranteed delivery/order (for streaming, games).
    • A simpler, connectionless Internet protocol in which error-checking and recovery services are not attempted. UDP is used when speed is desirable and error correction is not necessary, such as for live video streaming or online games.
  • DNS (Domain Name System): Translates human-readable domain names (e.g., `google.com`) into IP addresses. Hierarchical system.
    • The Internet's system for converting alphabetic domain names (like `www.example.com`) into numerical IP addresses (like `93.184.216.34`) that computers use to connect to each other. It's a hierarchical and distributed naming system.
  • HTTP (HyperText Transfer Protocol): Protocol for the World Wide Web (fetching web pages). HTTPS is the secure version using SSL/TLS.
    • HTTP: The underlying protocol used by the World Wide Web for transmitting hypermedia documents, such as HTML. HTTPS (HTTP Secure): An extension of HTTP used for secure communication over a computer network, encrypted using Transport Layer Security (TLS) or Secure Sockets Layer (SSL).
  • SMTP (Simple Mail Transfer Protocol): For sending emails.
    • An Internet standard communication protocol for electronic mail transmission.
  • SSL/TLS (Secure Sockets Layer/Transport Layer Security): Protocols for encryption and security, providing a secure channel for HTTP (making it HTTPS).
    • Cryptographic protocols designed to provide communications security over a computer network. TLS is the successor to SSL. They are widely used in applications such as web browsing (HTTPS), email, instant messaging, and VoIP.

Design Principles

Hierarchy: Seen in IP addressing and DNS, allowing scalability by distributing responsibility.

A system or organization in which elements are arranged in levels of subordination or ranking. In the Internet, DNS and IP addressing are hierarchical, which helps with organization and scalability.

Redundancy & Fault Tolerance: Multiple paths for data mean the Internet can withstand failures. If one link breaks, data can reroute.

Redundancy: The inclusion of extra components that are not strictly necessary to functioning, in case of failure in other components. Fault Tolerance: The property that enables a system to continue operating properly in the event of the failure of some of its components. The Internet's redundant pathways contribute to its fault tolerance.

Packet Switching: Data is broken into small Packets, each with header (address, sequence) and payload. Packets can take different routes and are reassembled at the destination. Efficiently uses network resources.

Packet Switching: A method of grouping data into packets that are transmitted over a digital network. Packets are routed independently and may take different paths to the destination, where they are reassembled. Packet: A small unit of data transmitted over a packet-switched network. Contains a header (with addressing, routing info) and a payload (actual data).

Scalability: The Internet's design (open protocols, hierarchical addressing, end-to-end architecture) allows it to grow to billions of devices. IPv6 was introduced to provide more addresses than IPv4.

The capability of a system, network, or process to handle a growing amount of work, or its potential to be enlarged to accommodate that growth. The Internet is designed to be highly scalable.

Cybersecurity

Protecting networks, devices, and data from unauthorized access or criminal use. Key concerns:

Vulnerabilities and Attacks:

  • Malware (Viruses, Worms, Trojans, Ransomware): Malicious software.
    • Malicious software designed to harm or exploit any programmable device, service or network. Includes viruses (attach to files, need user action), worms (self-replicate across networks), Trojans (disguise as legitimate software), ransomware (encrypts data demanding payment).
  • Phishing: Tricking users into revealing sensitive info via fake emails/websites.
    • A social engineering attack where fraudulent communications (often emails or fake websites) impersonate a trustworthy entity to trick individuals into revealing sensitive information like passwords, credit card numbers, or personal details.
  • DDoS (Distributed Denial of Service): Overwhelming a target with traffic from many compromised systems (botnet), making it unreachable.
    • An attack that floods a server or network with excessive requests/traffic from many compromised computer systems (often a botnet), aiming to make the target service unavailable to legitimate users.
  • Spoofing & MITM (Man-in-the-Middle): Intercepting/altering communications or impersonating systems.
  • SQL Injection / Software Vulnerabilities: Exploiting bugs in software (e.g., web forms) to manipulate systems or data.
  • Weak Passwords & Credential Reuse: Easily guessable or reused passwords are a major vulnerability.

Security Measures:

  • Encryption: Converting data into unreadable code (ciphertext) using a key.
    • Symmetric Encryption: Same key for encryption and decryption (e.g., AES). Key must be shared securely.
    • Asymmetric (Public-Key) Encryption: Pair of keys: public (shared openly) for encryption, private (kept secret) for decryption (e.g., RSA). Solves key exchange problem, used in SSL/TLS.
    • Encryption: The process of converting readable data (plaintext) into an unreadable format (ciphertext) using an algorithm and a key, so that only authorized parties with the correct key can decrypt and read it. Symmetric Encryption: Uses the same secret key for both encryption and decryption. Asymmetric (Public-Key) Encryption: Uses a pair of keys â€“ a public key for encryption (shared widely) and a private key for decryption (kept secret by the owner).
  • Digital Certificates & CAs (Certificate Authorities): CAs issue certificates to verify a public key belongs to a specific website (domain), enabling trust in HTTPS.
    • Digital Certificate: An electronic document used to prove the ownership of a public key, binding it to an entity (like a website or organization). Certificate Authority (CA): A trusted entity that issues digital certificates. Browsers and operating systems have a list of trusted CAs.
  • Firewalls: Filter network traffic, blocking unauthorized access.
    • A network security system (hardware or software) that monitors and controls incoming and outgoing network traffic based on predetermined security rules. It acts as a barrier between a trusted internal network and untrusted external networks (like the Internet).
  • Antivirus/Anti-malware Software: Detects and removes malicious software.
  • Software Updates/Patches: Close known security vulnerabilities.
  • Authentication (Passwords, Multi-Factor Authentication - MFA): Verifying user identity. MFA adds layers (e.g., something you know + something you have).
    • The process of verifying the identity of a user, process, or device. Common methods include passwords (something you know), security tokens or smart cards (something you have), and biometrics (something you are). Multi-Factor Authentication (MFA) uses two or more different factors.
  • User Education: Critical, as users can be a weak link (e.g., falling for phishing).

Bandwidth: Data transfer capacity of a network (bits per second). Latency: Time for data to travel source to destination (delay).

Bandwidth: The maximum rate of data transfer across a given path, typically measured in bits per second (bps). Latency: The delay before a transfer of data begins following an instruction for its transfer; the time it takes for a data packet to travel from source to destination.

Botnet: A network of private computers infected with malicious software and controlled as a group without the owners' knowledge, e.g., to send spam or launch DDoS attacks.

A network of computers infected with malware (bots) that are under the control of a single attacking party, known as the "bot-herder." Botnets are often used to launch DDoS attacks, send spam, or steal data.

Big Idea 7: Global Impact

Summary

Computing has profound global impacts â€“ beneficial and harmful â€“ on people and society. This Big Idea examines how computing innovations transform communication, learning, work, and life, and how societal factors influence these developments. It addresses issues like the digital divide, privacy, intellectual property, ethics, and bias.

Detailed Concepts

Computing and Communication

Computing enhances communication (email, chat, video conferencing) and collaboration (shared online tools, cloud computing). Social media fosters new interactions and online communities. Access to information (search engines, Wikipedia) is widespread. New forms of expression (blogs, podcasts, videos) empower individuals. Assistive Technologies enhance human capabilities.

Hardware and software designed to help people with disabilities perform tasks that might otherwise be difficult or impossible. Examples include screen readers for the visually impaired, speech-to-text software, and adaptive keyboards.

Problem Solving at Scale

  • Crowdsourcing: Obtaining input/work from many people via the Internet (e.g., Kickstarter, Wikipedia).
    • Obtaining information or input into a task or project by enlisting the services of a large number of people, either paid or unpaid, typically via the Internet.
  • Citizen Science: Public participation in scientific research (e.g., classifying galaxies, reporting bird sightings).
    • Scientific research conducted, in whole or in part, by amateur (or nonprofessional) scientists. Often involves public participation in data collection and analysis.
  • Human Computation: Using human intelligence for tasks computers find hard (e.g., CAPTCHAs, image tagging).

Innovation Across Fields

Computing drives innovation in medicine (diagnostics, bioinformatics), business (e-commerce, FinTech), science (simulations, genome sequencing), education (online learning), entertainment (CGI, games), etc. Open Access and Creative Commons enable broader sharing of information and creative works.

Open Access: A publishing model for scholarly communication that makes research information available to readers at no cost. Creative Commons: A non-profit organization that provides standardized licenses creators can use to grant public permission to share and use their creative work on conditions of their choice.

Beneficial vs. Harmful Effects

Computing innovations have dual impacts:

  • Beneficial: Easier access to info/education, new jobs, social connectivity, efficiency, medical advances, creativity.
  • Harmful:
    • Privacy Invasion: Collection and aggregation of personal data (location, browsing habits) by apps, websites, IoT devices.
      • The intrusion into the personal life of another, causing outrage, mental suffering, or humiliation to a person of reasonable sensitivities. In computing, this often relates to the unauthorized collection, use, or disclosure of personal data.
    • Security Risks: Cyber threats disrupting infrastructure, identity theft.
    • Intellectual Property (IP) Issues: Easy copying/sharing challenges copyright. DMCA (Digital Millennium Copyright Act) addresses this.
      • Intellectual Property: Creations of the mind, such as inventions; literary and artistic works; designs; and symbols, names and images used in commerce. IP rights include copyrights, patents, and trademarks. DMCA: A 1998 United States copyright law that criminalizes production and dissemination of technology, devices, or services intended to circumvent measures that control access to copyrighted works.
    • Social Issues: Misinformation/propaganda, echo chambers, internet addiction, mental health impacts, Job Displacement due to automation.
      • The elimination of jobs that occurs when technological advancements, automation, or economic shifts make certain roles obsolete or reduce the need for human labor in those positions.
    • Environmental Impact: E-waste, energy consumption of data centers/cryptocurrency.
    • Ethical Concerns: Bias in Algorithms (AI reflecting societal biases), surveillance, Internet Censorship.
      • Ethical Concerns: Issues that involve considerations of right and wrong, fairness, and responsibility in the context of computing. Bias in Algorithms: Systematic and repeatable errors in a computer system that create unfair outcomes, such as privileging one arbitrary group of users over others, often stemming from biased training data or flawed assumptions in the algorithm design. Internet Censorship: The control or suppression of what can be accessed, published, or viewed on the Internet, often by governments or organizations.

The Digital Divide

Unequal access to computing and the Internet due to socioeconomic status, geography, education, etc. It can exacerbate existing inequalities. Efforts to bridge it include cheaper tech and educational programs.

Digital Divide: The gap between demographics and regions that have access to modern information and communications technology (ICT), and those that don't or have restricted access. The gap between those who have ready access to computers and the Internet, and those who do not. This can exist between countries, regions, socioeconomic groups, or even neighborhoods, and affects opportunities in education, employment, and civic participation.

Context Matters

Innovations influence and are influenced by their social, economic, and cultural contexts. Society's needs drive tech, and tech changes society. Impacts vary globally (e.g., mobile banking in regions with few traditional banks).

Other Key Terms

Innovation (Computing): A new or significantly improved technology, product, process, or idea that includes a computer or program as a central part of its function. Open Source Software: Software whose source code is publicly available for anyone to inspect, modify, and distribute, fostering collaboration. Software for which the original source code is made freely available and may be redistributed and modified. Examples include Linux and Apache Web Server.

Example: Social Media

Purpose: Connect people, share content. Beneficial: Rapid info dissemination, social movements, business advertising. Harmful: Misinformation, cyberbullying, privacy issues, data harvesting. Context: Drives new job roles (influencers), shifts ad revenue, faces varying cultural acceptance and censorship globally. Raises ethical debates on content moderation and data use.
